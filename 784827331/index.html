什么是联合类型

联合类型（Union Type）可以通过管道符 | 将变量设置多种类型。

比如最常见的，你可能在你的代码中这么写过：let value: string | number，这其实是就是一个简单的联合类型，它允许为一个变量设置多种可能类型。

这里我讲解以下特殊的场景。

联合类型在判断中的体现

先看下面的代码：

interface Square {
  kind: 'square'
  size: number
}
interface Rectangle {
  kind: 'rectangle'
  width: number
  height: number
}

type Shape = Square | Rectangle

这里分别定义了 Square 和 Rectangle 两个类型接口，同时定义了一个 Shape 联合类型。

在一些 if...else 判断中，联合类型的好处就体现出来了。

function area(shape: Shape) {
  if (shape.kind === 'square') {
    // 这里 TS 会推断出参数 shape 为 Square
    // 所以，这里你的 shapre 对象只能取到 Square 接口上定义的属性
    // shape.width 可以安全调用，因为 Square 接口上已经定义了该属性
    return shape.width
  } else {
    // 在 else 中，TS 会推断出 shape 为 Rectangle
    return shape.width * shape.height
  }
}

在 VSCODE 中，鼠标放在 else 条件中的 shape 对象上，可以看到，TS 正确推断出了 shape 的类型：

联合类型的强大和美妙之处就在于此，但远不止于此。

更加详细的推断

我们依旧使用前定义的 Shape 类型，但是对 area 函数做一些修改：

function area(shape: Shape) {
  if (shape.kind === 'square') {
    return shape.width
  }
  // 我们删除了 else，即抛弃了当 shape 类型为 Rectangle 的时候的处理
}

我在这里，我不小心忘记处理了 shape 类型为 Rectangle 的情况。

如何解决这种问题呢？

我们可以利用简单的向下思想，来确保块中的类型被推断为与 never 类型兼容的类型。

举个 🌰。

function area(shape: Shape) {
  if (shape.kind === 'square') {
    return shape.width
  } else if(shape.kind === 'rectangle') {
    // 由于后面 else 条件中的代码，导致你必须添加该处理
    return shape.width * shape.height
  }else {
    // 通过定义一个将 shape 赋值给另外一个变量，并且将变量定义为 never；
    // 这将强制你去处理每一个边界情况
    const _exhaustiveCheck: never = shape
    return _exhaustiveCheck
  }
}

在编辑器中，如果你不小心忽略了对某个类型的处理，那么 TS 会立即提示你：

该约束同样适用于 switch 语法。

strictNullChecks

如果你 tsconfig.json 文件中使用了 strictNullChecks，不要忘记返回 _exhaustiveCheck 变量，否则 TS 可能会推断返回值为 undefined。

总结

  1. 联合类型通过管道符来实现；
  2. 联合类型可以在一些场景下（比如 if...else）自动推断出当前类型；
  3. 将联合类型的对象赋值给 never 类型的对象，可以避免遗漏一些条件判断；
