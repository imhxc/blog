新瓶装旧酒，从另一个角度去理解 JavaScript 面向对象

前言

这篇文章所涉及的知识基本上是我早就掌握了的，但是这次我尝试带着问题，从另一个角度去学习、探索 「JavaScript 为什么是面向对象语言」？

两个问题

在这之前，我们先明确两个问题。JavaScript 中，有许许多多的对象，但是在 ES6 之前，JavaScript 没有类的概念。

即使到了 ES6，类的实现也仅仅是一个语法糖，ES6 中的类的绝大部分功能，ES5 都可以做到。新的 class 也只是让对象原型的写法更加清晰、更像面向对象编程的语法而已。

另外，JavaScript 语言相较于 Java 等语言来说，是比较自由的（PHP 也很自由），可以任意、随时的添加属性，修改对象。

所以，我们知道了上面提到的两种现象后，我们要去探索「Why」。

  • 为什么 JavaScript （直到 ES6）有对象的概念，但是却没有像其他的语言那样，有类的概念呢？
  • 为什么 JavaScript 对象里可以自由添加属性，而其语言却不能呢？

JavaScript 标准对基于对象的定义是：语言和宿主的基础设施由对象来提供，并且 JavaScript 程序即是一系列互相通讯的对象集合。

什么是面向对象

在理解「对象」这个词之前，我们需要先理解「对象」的自然定义。

很简单，「对象」的自然定义可以有：

  1. 一个可以触摸或者可以看见的东西；
  2. 人的智力可以理解的东西；
  3. 可以指导思考或行动（进行想象或施加动作）的东西；

有了自然定义之后，我们在来看在不同编程语言中，「对象」的使用。

最为常见的，是使用「类」的方式来描述编程语言中的「对象」，如 Java、C++ 等。

但是，JavaScript 却”剑走偏锋”，选择了「原型」来实现。

由于 JavaScript 最初是模仿的 Java，这导致 JavaScript 创始人 Brendan Eich 在原型运行时的基础上引入了 new、this 等语言特性，这样看起来更像 Java。

因为「原型」的设计，导致大量 JavaScript 程序员开始围绕原型去模仿、实现「类」，也进而产生了一些框架，如 PrototypeJS、Dojo 等。

JavaScript 对象的特征

不论使用什么样的编程语言，「对象」都有一些本质特征（参考 Grandy Booch《面向对象分析与设计》）。

  • 对象具有唯一标识性：即使完全相同的两个对象，也并非同一个对象；
  • 对象具有状态，同一对象可能处于不同状态之下；
  • 对象具有行为：即对象的状态，可能因为他的行为产生变迁。

对象具有唯一标识性

一般而言，各种语言的对象唯一标识性都是用内存地址来体现的，对象具有唯一标识的内存地址，所以具有唯一标识。

状态、行为

在不同的语言中，会有不同的术语来描述「状态」和「行为」。

比如在 C++ 中，他们分别称为「成员变量」和「成员函数」；在 Java 中他们分别为「属性」和「方法」。

但是在 JavaScript 中，把「状态」和「行为」统一抽象为了「属性」，下面这段常见的代码就体现了这一点：

const obj = {
  name: 'Isaac',
  sayHi() => {
    alert('Hi!');
  }
}

在实现了对象基本的特征的基础上，JavaScript 中对象独具特色：对象具有高度的动态性，这是因为 JavaScript 赋予了使用者在运行时为对象添改状态的行为的能力。

另外，JavaScript 的恶属性被设计成比其他的语言更加复杂的形势，它提供了数据属性和访问器属性（getter/setter）两类。

JavaScript 对象的两类属性

对于 JavaScript 来说，属性并非只是简单的名称和值，它用一组特征（attribute）来描述属性（prototype）。

数据属性

前面提到了，属性也分为数据属性和访问器属性。这里先来介绍下数据属性。

数据属性比较接近于其他语言的属性概念，其具有四个特征：

  • vlaue: 属性值；
  • writable：属性是否可写；
  • enumerable：当前属性是否可被 for in 枚举；
  • configurable：当前属性是否可被配置；

访问器属性（getter/setter）

访问器属性也有四个特征：

  • getter：函数或 undefined，在取属性值时被调用；
  • setter：函数或 undefined，在设置属性时被调用；
  • enumerable：决定 for in 能够枚举该属性；
  • configurable：决定该属性能否被删除或者改变特征值；

实际上，JavaScript 对象的运行时是一个「属性的集合」，属性以字符串或者 Symbol 为 key，以数据属性特征值或者访问器属性特征值为 value。

对象是一个属性的索引结构（索引结构是一类常见的数据结构，我们可以把它理解为一个能够以比较快的速度用 key 来查找 value 的字典）。

我们以下面代码为例：

const obj = {
  get a() {
    return 1;
  }
}
obj.a; // 1

在这里，a 是 key，{ writable: true, value: 1, configurable: true, enumerable: true } 是 value。

到这里，也可以解释「JavaScript 不是面向对象」的这种说法了，因为 JavaScript 的对象设计跟主流基于类的面向对象差异非常大。

但是同时，JavaScript 又提供了完全运行时的对象系统，使得它可以模仿多数面向对象编程范式，所以，在这一点上，也可以认为「JavaScript 也是面向对象语言」。

同时，JavaScript 语言标准也已经明确说明，JavaScript 是一门面向对象语言。
