JS原生Ajax探析(XMLHttpRequest)

发表于 2018-01-19 | 分类于 阅读笔记 |
Ajax的核心技术就是XMLHttpRequest(XHR)对象, 是由微软最先引入的, 也就是说, 最先实现的是IE5浏览器。 Ajax实现后, 引发的了一系列的技术发展和问题, 如跨域问题的解决各种方法、XHR2级的实现以及后面的长短轮询, 实现服务器推送的的方式、SSE等。最新的WebSocket双工、双向通信技术也应运而生。

XMLHttpRequest对象

前面提到, IE5是最早引入XHR对象的浏览器。但是实现方式和和后来的其他浏览器有一些区别。在IE5中, XHR对象是通过MSXML库中的一个ActiveX对象实现的。所以, 在IE版本中会有三种不同版本的XHR对象, 即MSXML2.XMLHttp、MSXML2.Http.3.0和MSXML2.XMLHttp.6.0。所以这一兼容问题, 根据实际开发需求, 也是需要我们酌情处理的。在IE7+和其他浏览器都支持原生的XHR对象。创建XHR对象需要使用XMLHttpRequest对象。

创建XHR对象:

🌰 var xhr = new XMLHttpRequest();

XHR的用法

创建XHR前面已经介绍过, 即使用XMLHttpRequest构造函数new一个对象。

🌰 var xhr = new XMLHttpRequest();。

XHR对象有两个基本的方法:

  • open();
  • send();

顾名思义, 一个是打开, 一个是发送。我们一个一个来看。

open():

open()接收3个参数: 1.要发送的请求类型(‘GET’、’POST’等); 2.请求的URL; 3.是否发送异步请求的布尔值。需要注意的是, 第三个参数, 一般的都是true, 如果使用false作为同步操作, 后续代码将无法立即执行。

🌰 xhr.open('GET', 'baidu.com', true);。

send():

send()接收1个参数, 即要作为请求主体发送的数据。需要注意的是, 如果没有需要作为请求主体发送的数据, 建议传递一个null, 可以兼容有些浏览器对该参数的必传的一个要求。

🌰 xhr.send(null);。

在使用send()方法后, 浏览器就会向服务器发送一个请求, 得到响应后, 响应的数据会自动的填充XHR对象的相关属性。常用属性如下:

  • responseText: 作为响应主体被返回的文本。
  • responseXML: 作为响应主体被返回的XML DOM文档。需要设置相应的内容类型是text/xml或application/xml。
  • status: 响应的HTTP状态, 如200, 304, 404。
  • statusText: HTTP状态的说明。
  • readyState: 标识XHR对象当前处于哪种状态。

对于上述几种属性, 最重要的需要我们关注的是readyState属性, 因为我们需要监听该属性, 来判断当前XHR对象处于那种状态, 以确保我们可以正确的处理数据。

readyState属性有5中值:

  • 0: 未初始化。即尚未调用open()方法。
  • 1: 启动。已经调用open()方法, 但是还没有调用send()方法。
  • 2: 发送。已经调用send()方法, 但是尚未接收到响应。
  • 3: 接收。已经接收到部分数据。即响应头已经接收到。
  • 4: 完成。已经接收到全部数据。即响应体已经接收到。

abort()

🌰 xhr.abort();。

该方法可以终止XHR对象, 而且也不允许访问任何与响应有关的对象属性, 并且readyState也会被设置为0。建议终止请求后, 还应该对XHR对象进行解除引用。

XMLHttpRequest2级

前面的XMLHttpRequest是1级规范。XMLHttpRequest2级则更进步一的丰富了XHR对象。但是需要注意的是, 并非所有的浏览器都实现了XMLHttpRequest2级规范, 但是所有的浏览器都实现其部分内容。下面我们来一一介绍其相关属性。

FormData

前面的, 我们曾经使用POST模拟发送了一个表单对象, 如果我们表单数据量多的话, 序列化会显得比较繁琐。为此, XMLHttpRequest2级为此定义了一个FormData类型。就是用来序列化表单对象, 便于XHR的传输。

使用方法:

new 一个FormData对象即可。

如果你觉得麻烦, 可以直接向FormData构造函数传入一个表单元素, 会自动的根据表单元素里的一些标签上的name属性来填充对应的值。

var data = new FormData(document.form[0]);。

然后我们可以直接将序列化好的数据传给send():

xhr.send(data);。

🎉 使用FormData的方便之处在于不必明确的给XHR对象设置响应的请求头, XHR对象能够自动识别传入的数据类型是FormData的实例, 并配置适当的头部信息。

⚠️ 支持FormData的浏览器有 Firefox4+、Safari、Chrome、Android3+版WebKit。

overrideMimeType()方法

overrideMimeType()方法用于重写XHR响应的MIME类型。因为响应的MIME类型决定了XHR对象如何处理它, 所以我们需要根据实际场景来修改MIME类型。

例如, 服务器返回的MIME类型是text/plain, 但实际数据中包含的是XML。由于MIME类型的限制, 即使数据是XML类型, responseXML属性中仍然是null。

通过调用overrideMimeType()方法, 可以保证把响应数据当作XML而非纯文本来处理。

进度事件

Progress Events 可以方便的让我们来处理一些大数据上传时候等待的情况。

有以下6个进度事件:

  • loadstart: 在接收到响应数据的第一个字节时触发。
  • progress: 在接收响应数据期间持续不断的触发。
  • error: 在请求发生错误的时候触发。
  • abort: 在因为调用abort()方法而终止时触发。
  • load: 在接收到完整的响应数据时触发。
  • loadend: 在通信完成或者触发error、abort或load事件后触发。

每个请求都从触发loadstart事件开始, 接下来是一个或多个progress事件, 然后触发error、abort或load事件中的一个, 最后以触发loadend事件而结束。

JSONP

JSONP和JSON差不多, 只不过是被包含在了回调函数里。

🌰 cllback({name: 'hxc'});。

JSONP 由两部分组成: 1.回调函数; 2.数据。

回调函数是当响应到来的时候应该在当前页面中调用的函数。回调函数的名字一般是在请求中指定的。而数据就是传入回调函数的JSON数据。

Comet

Ajax是一种从页面向服务器请求数据的技术, 而Comet则是一种服务器向页面推送的技术。Comet能够让信息近乎实时的被推送到页面上, 非常适合处理体育赛事和股票报价的场景。
有两种实现Comet的方式: 1.长轮询; 2.流。

长轮询

长轮询是比较传统轮询(即短轮询)的一个翻版。传统轮询即不停的向服务器发起请求, 以获取最新的数据。而长轮询则是当浏览器发起一个请求后, 服务器一直保持链接, 当有数据可发送, 服务器发送数据并关闭链接, 随机浏览器接收到数据后在重新发起一次请求重复上述操作。

流

流作为实现Comet的第二种方式, 其实是实现HTTP流。流不同与上述的两种轮询, 因为它在整个页面的声明周期只使用一个HTTP连接。具体的实现方式就是: 浏览器向服务器发起一个请求, 而服务器则保持连接打开, 然后周期性的向浏览器发送数据, 最关键的地方在于, 当服务器每次向浏览器发送数据的时候, readyState属性都会改变为3, 这样我们就可以来监听readyState来获取新的数据, 实现服务器的消息推送。

SSE(Server-Sent Events)

SSE, 服务器发送事件, 也属于Comet交互的一种模式。用于创建到服务器的单向链接, 服务器通过这个连接可以发送任意数量的数据。但是, 需要注意是, 服务器响应的MIME类型必须是text/event-stream, 而且是浏览器中JavaScriptAPI能够解析的格式输出。SSE支持短轮询、长轮询和HTTP流, 而且能够在断开连接时自动确定何时重新连接。

使用方法
