正则表达式(一)

发表于 2018-03-17 |
正则表达式是一种工具，无论我们是前端还是后段开发者，都是程序员必须掌握的一种基本技能。

重复匹配

我们先尝试一下如何匹配一个邮箱：
505421473@qq.com

熟悉正则一些基本语法的人都应该多多少少会知道如何去做，可能会写一个如下的正则表达式:
\w@\w.\w

可惜的是，上面的正则表达式有一些局限性，虽然对505421473@qq.com可以成功匹配, 或者说只能匹配形如a@b.c的邮箱，如果我们改成hxc@163.cn.com

匹配一个或者多个字符（+）

要想匹配一个或者多个字符，也就说重复匹配，我们只需要简单的给这个字符（或者字符集）加上一个+字符作为后缀即可。

我们来匹配一个邮箱:505421473@qq.com
正则表达式: w+@\w+\.\w+

/w+@\w+\.\w+/.test('505421473@qq.com'); // true

另外，如果我们在一段文本中区匹配邮箱，也是可以的:

这里需要注意的是，在js中使用text.match(regexp)的时候，只会返回第一个邮箱:ben@forta.com，因为我们的正则表达式并没有执行全局搜索，如果需要全局匹配，需要添加g标识符，只需要将regexp的表达式改为:/w+@\w+\.\w+/g即可

这里我们介绍了+字符，另外还有*和?操作符，意思分别为：*: 匹配0或者多次,?: 匹配0或者一次。用法和+一样，这里就不过多介绍。

我们再来思考下我们之前的邮箱地址和正则表达式:
邮箱地址:505421473@qq.com
正则表达式: w+@\w+\.\w+

在这种情况下，是完美的。但是我们要知道的是, 有一些邮箱，比如:505421473@qq.com.cn这种形式也是合法的。显然，我们当前的正则表达式无法匹配到。

问题就在于@字符后会存在多个点(.)

解决办法：我们要了解的，邮箱中的点(.)字符前面是一定要跟一些英文或者数字的，换句话说，英文(或数字)和点(.)字符是一个组合，应该成对出现的。了解了这一规则后，很明显，我们可以使用字符集合([])的方式来解决。

修改后的正则: [\w.]+@[\w.]+\.\w+
测试邮箱：a.a.a@b.b.com
/[\w.]+@[\w.]+\.\w+/.test('a.a.a@b.b.com'); //true

另外我们发现，在字符集中，我们没有对.字符进行转义，尽管如此，我们的语法依然是有效的。因为一般来说，在字符集中，像.和+这样的元字符会被解析为普通字符，不需要转义。

对于前面我们修改后的正则仍然是有问题的，如果.a.a@b.b.com这样的邮箱也会匹配到，问题就出现在开头的.字符。这里就需要用到*这个元字符。

再次优化：^\w+[\w.]*@[\w.]+\.\w+。

我们通过^字符来限制开头必须是英文或者数字，出现一次或者多次，然后我们通过[\w.]*来匹配一些形如@前面的一些字符，限制它可以出现零次或者多次。

当然，我们再次优化后的邮箱匹配仍然是有问题的，比如a..a@.com仍然可以匹配成功。

匹配的重复次数

前面我们提到了+、*、?元字符，但是如果我们需要自定义匹配次数，比如我们只想匹配3到6次该怎么办？

这里，我们可以使用{}元字符。使用很简单，比如我们像匹配3到6次:{3,6}, 也就是说至少匹配3次，最多重复匹配6次。

另外，如果我们只写一个数字:{3}，意为仅匹配3次。

如果我们想匹配3到多次: {3,}。

回溯引用

所谓回溯引用，通俗的来说，就是继续使用前一个规则，来匹配后面的内容。

什么是回溯引用，我们直接来看一个例子：比如我们需要匹配一段HTML代码，找出h1到h6以及其配对标签。

这里，我们需要把所有的标题标签(h1-h6)查找出来，熟悉子表达式的话，或许我们会写出如下正则:
<h[1-6]>.*?<\/h[1-6]>

很明显，我们的标签语法是错误的，只有<h6>标签是正确的，如果我们还是使用前面的正则区匹配，明显会把一些错误的语法标签也匹配出来。

这时候，就需要我哦们的回溯引用出场了。

使用回溯引用进行优化

这里，我们使用回溯引用来解决前面的问题。

我们先来看最终的正则:<h[(1-6)]>.*?<\/h\1>。

我们挨个来分析这段正则。

  • <h([1-6])>.*?: 这段正则应该都很清楚，就是匹配<h1>到<h6>这些开始标签的，以及匹配标题标签之间的内容:.*。另外()代表的意思是包含着一个子表达式;
  • <\/h\1>: 这段正则我们先不看\1这段。其余的便是去匹配结束标签。

我们重点来看\1。很明显，是这段正则发挥了作用。这个便是回溯引用。\1引用的正是前面的那个表达式()。换句话说，如果前面匹配到的是6， 那么，这里的\1就会替换成6， 那么正则就会变成这样:<h6>.*?<\/h6>(写成这样的形式只是为了理解)。

或许会有疑问，为什么可以准确的指向[1-6],而不只想其他的？

之所以这样，是因为我们对[1-6]用了一个()来包裹起来，这就代表[1-6]是一个子表达式，而\1中的1的意思就代表指向第一个子表达式，很明显这里的[1-6]就是第一个子表达式。

同样我们可以用\2,\3来只想第二个、第三个子表达式。

再来一个小训练

这里，我们来查找一段文本中重复出现两次的单词。

上边一段文本中，重复出现了of、are和and单词。

我们使用一下回溯引用来查找出来。

[ ]+(\w+)[ ]+\1

这里我们使用[ ]+来匹配空格，使用\w来匹配单词,并且使用()包裹成一个子表达式方便后面的\1进行回溯引用。换句话说，如果我们的\w匹配到of，那么接着，后面的我们就还要去匹配of。

总结

这里，介绍了如果去限制正则中匹配的次数，正则表达式为我们提供了3中元素符:

  • +: 匹配1到多次
  • *: 匹配0到多次
  • ?: 匹配0到1次

另外如果我们需要自定义匹配出现的重复次数，可以使用{}元字符来设置。

除了可以去控制匹配次数外，我们了解到如何在一些特殊的场景下去使用回溯引用。回溯引用通过\1的形式去引用(指向)前面的第一个子表达式。同样的，如果需要引用第二个，则写成\2即可。

在日常开发中，对于需要使用到正则的地方，我们首先需要去了解需要匹配的字符串的规则，如此，才能写出对应的正则。
