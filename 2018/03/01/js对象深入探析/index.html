js对象深入探析

发表于 2018-03-01 |
对象并无复杂的地方，拿来即用。但是如果我们深入去研究，会发现对象也有很多的知识点，很多需要注意学习的特性，以及在每个场景下我们又需要注意什么。

这篇笔记整理来源于《你不知道的JavaScript》和《JavaScript权威指南》这两本书的对象相关章节。欢迎大家支持正版，尊重知识。

基本语法

我们都知道对象有两种基本的定义方式: 声明(字面量)形式和构造(new)形式。
需要注意的是, 通过Object.create()也可以创建一个对象。 不过严格意义上来说, 使用这种方法是为了给新创建的对象指定的原型对象和属性, 由于目的不同, 所以我个人不喜欢将Object.create方法归纳到定义对象方法中。

需要注意的是, 通过Object.create()方法创建对象, 必须传递一个参数: 需要继承的原型, 如果没有, 则传递null。所以一般的， 如果我们需要继承某个对象的时候，可以选择这种方法。

声明形式和构造形式本质上没有什么异同, 如果非要说有什么区别，也就是在添加属性的时候有些不同, 声明形式可以在创建的时候就可以添加多个键值对, 但是构造形式必须逐个添加。

类型

对象是JavaScript的基础。在JavaScript中一共有六种数据类型，加上ES6新增的Symbol，一共7种。

  • string
  • number
  • boolean
  • null
  • undefined
  • symbol
  • object

需要注意的是, 这里只得不是JS中的内置对象, 所以我们这里用了小写形式来描述, 了解过包装对象的应该很清楚这种区别。

在这六种类型中, string、number、boolean、null、undefined、symbol本身并不是对象, 都属于简单基本类型。在这里, 我们会有一个误解: null是一个对象类型。因为执行typeof null后会返回object形式的字符串。但是这里要告诉大家, 这是一个BUG, 或者说是历史遗留问题。

不同的对象在底层都表示为二进制, 在JavaScript中二进制前三位都为0的话会被判定为object类型, null的二进制表示全是0, 自然前三位也是0, 所以执行typeof会返回object字符串。

在后来的W3C工作草案中, 曾经提出要修改这一问题, 让typeof null返回正确的值, 可惜这样会导致所有运行的网址出现严重的问题, 所以就将错就错了。(从网上道听途说的)

另外， 还有一种说话也是需要纠正的: JavaScript中一切皆对象。, 这明显是错误的。

或许你会奇怪, 为什么没有array和function类型，实际上，这两种都属于对象子类型, 或者叫复杂的基本类型。特别是数组, 在我早前的一篇笔记中曾经写过”为什么有些自定义的键值对并不能算是数组元素“，并且写的比较混乱，现在回过头来看， 有很多理解不当甚至错误的地方。通过后面的介绍, 会明白许多。

内置对象

  • String
  • Number
  • Boolean
  • Object
  • Function
  • Array
  • RegExp
  • Date
  • Error

上述的几种对象都是由JS提供给我们的, 不同于前面的数据类型概念, 这里的内置对象可以理解为是一种可以操作调用的函数(例如使用new操作符), 所以一定要区分, 虽然名字的表示一样, 但是叫法和概念完全不一样。在《JavaScript高级程序设计》这本书中, 有一章“引用类型”做了讲解。

对象的操作

对象的内容是由一些存储在特定位置（任意类型）的值组成的, 我们称之为属性, 其中属性又具有属性名和属性值。其实在JS引擎中, 我们定义一个变量，并且赋值，通常这个变量只是一个指向该值的存储位置的指针。同样的，在对象内部也是这样, 对象内部存储的属性，也仅仅是一个指向具体值的指针，也就是属性名。

复制对象

对于复制对象与之相关的便是引用。

对于复制有浅复制和深复制之分，又叫浅拷贝和深拷贝。我们需要知道的是, 如果完全的深拷贝有可能会导致一个死循环。

一般的, 我们如果想要复制一个对象， 有一种方法: JSON.stringify():
let newObj = JSON.parse(JSON.stringify(obj));

当然，这种方法显然是不好的，在ES6中给我们提供了新的方法: Object.assign()。该方法接收多个参数, 第一个为目标对象, 之后可以传一个或者多个源对象。会遍历一个或者多个源对象的所有可枚举的自有键(非原型链上的), 最后返回目标对象。

Object.assign;

注意，以上两种方法都是浅复制。

需要注意的是: 由于Object.assign使用的是=操作符来赋值，所以源对象中的一些属性描述符不会被复制到目标对象中。

属性描述符

对象的属性除了具有属性值, 还有其他另外3个特性, 通过自定义这三个特性, 我们可以定一个类似ES6中通过const关键字定义的常量，以及其他需求。

首先我们需要知道都有哪三个特性:

  • writable: 决定是否可修改属性的值
  • enumerable: 决定该属性是否可枚举
  • configurable: 决定该属性是否可以配置

上面三个属性默认都是true。

Object.getOwnPropertyDescriptor()方法可以用来获取一个属性的描述符。接受两个参数，分别是目标对象和目标对象中的属性名，存在则返回一个属性描述对象，否则返回undefined。

同样的， 如果我们尝试遍历该对象上的属性的时候,会跳过该属性, 因为我们对该属性设置不可枚举: enumerable: false。如果我们再次使用Object.defineProperty()来修改name属性的特性的时候, 也会失败, 并且直接报错。

需要注意的以下几点:

  • configurable是单向操作，也就是说当设置为false的时候，不能在重新设置为true。
  • configurable为false不可配置的时候, 有一个列外: 当writable为true的时候，可以设置为false, 但是反过来从false设置为true就不行。
  • 在严格模式下use strict，当writable为false的时候，再次赋值会报错。
  • 当configurable为false的时候, 还会禁止使用delete删除这个属性。

禁止扩展:Object.preventExtensions()

Object.preventExtensions()方法可以禁止一个对象添加新属性并且保留已有的属性。

密封:Object.seal()

Object.seal()会创建一个“密封”对象。需要知道的是，这个方法实际上会在一个现有对象上调用Object.preventExtensions()并把所有的现有属性标记为configurable: false。

密封的意思就是：不能添加新属性，也不能重新配置属性描述符，但是可以修改属性值。

冻结:Object.freeze()

Object.freeze()会创建一个冻结对象，这个方法实际上会在一个现有对象上调用Object.seal()并把所有属性的writable特性修改问题false。

冻结的意思就是：不能添加新属性，不能重新配置属性描述符，也不能修改属性值。

当然，这和”浅复制”一样，是一个”浅冻结”，意思就是我们依然可以修改对象中属性所指向的那个内容，换一种说法就是依然可以修改所引用的那个对象。

总结

对象的相关知识还有很多，还需要慢慢去探索。

  • “JavaScript一切皆对象”的观点是错误的。
  • 对象就是键值对的集合, 但是属性不一定包含值, 如getter/setter的访问描述符。
  • 数组、函数也属于对象中的子类型。并且数组是一中比较特殊的对象，特殊在于数据结构上不同于普通对象，并且对于数组的索引是有合法性限制的。
  • 属性值的读取其实是[[Get]]、[[Put]]这两种操作， 对应Getter和Setter，并且我们可以通过显示的设置Getter和Setter来实现数据的双向绑定。
  • 通过自定义属性描述符我们可以设置一个常量，但是需要搞清楚常量的概念：对于常量我们仍然可以修改这个指针所指向(或者叫引用)的值，只不过无法替换这个常量的指向(引用)对象。对于一些浅冻结的概念也是如此。
  • Object.preventExtensions()、Object.seal()、Object.freeze(), 这三个方法对于属性的操作限制逐步增加, 分别为: 禁止添加新属性; 禁止添加新属性和配置特性; 禁止添加新属性、配置特性以及修改属性值, 可以理解为是一种语法糖。
